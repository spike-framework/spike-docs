Spike Compiler

Spike Compiler was created to add some features as Imports, GStrings and powerful templates engine and more in the future.
Compiler is written in Groovy, so have to be executed with Java Virtual Machine.

Grunt shell integration

Warning! Please be carefully with usage grunt-shell because it hangs very ofter

With grunt, it's recommended to use grunt-shell-spawn plugin which offers synchronous execution of shell tasks.

Example integration compiler in Gruntfile.js 


                ...

                shell: {
                    compileImportsAndGStrings: {
                        command: 'java -jar bower_components/spike-compiler/build/libs/spike-compiler.jar imports-gstrings  dist/js/app.js dist/js/app.js'
                    },
                    compileTemplates: {
                        command: 'java -jar bower_components/spike-compiler/build/libs/spike-compiler.jar templates  src/app dist/js/templates.js'
                    }
                },

                ...

Options

imports-gstrings
Compiler option to process imports and gstrings syntax in given file and output compiled file to given path Example: 
                        java spike-compiler.jar -jar imports-gstrings dist/js/app.js dist/js/app.js
templates
Compiler option to find out and process whole templates files in given directory and output compiled templates into single file to given path Example: 
                        java spike-compiler.jar -jar templates  src/app dist/js/templates.js
