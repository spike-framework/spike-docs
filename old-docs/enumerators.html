Enumerator module

Spike Framework divide application into separately modules and one of them is enumerator.

enumerator should be used as storage for constants represented as primitives as well as simple functions processing those constants.

Enumerators are singletons

Enumerators preferable directory structure should be as follows:

  • src/app (value from app.config.rootPath)
    • enum
      • status.enum.js
      • response.enum.js

Also naming convention should be keeped as:

Enumerator file name should be named ENUMERATOR_NAME.enum.js

Example declaration of enumerator:

                'import $this as app.enumerator.Status';

                app.enumerator.register("Status", {

                    OK: 200,
                    NOT_FOUND: 404,
                    FORBIDDEN: 403,
                    INTERNAL_SERVER_ERROR: 500

                });

Then enumerator can be used with static reference as well in imports in any another modules: 

               'import $statusEnum as app.enumerator.Status'; //import way

                app.enumerator.Status //static way
