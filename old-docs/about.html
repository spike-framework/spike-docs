History

Spike Framework in general was build to handle cordova mobile applications with lightweight and extendable structure
written in plain JavaScript and with support jQuery usage.

Applications had to run on old devices as well as on new ones, so I decided to avoid any kind of modern frameworks as Angular or Ionic with Angular modified core,
Knockout, React, Reactive etc. because very often those frameworks generated more problems, than they resolved.

Also the problem with modern and big frameworks is enormous documentation (mainly for Angular and Ionic) which can take big percentage of application development process
to learn and study framework details.

Another topic is TypeScript usage, which I think is nice feature but in general it also create more problems such as longer project build time, only shallow OOP usage because of
dependencies, which in most cases are not typed and even they are typed, it takes long time to learn tens of new classes and interfaces.
In general, using TypeScript slows down development process and for me - takes away enjoyment of coding in JavaScript

Now

Spike Framework after almost two years of internal use in some companies was released in 2017 (resources was always open-source and available since 2016).

Only contributor is Dawid Senko as the author but if somebody reading this wants to join community and help with development, documentation, testing
or any other proposes please contact me dsenko26@gmail.com or via GitHub message

Short summary

Spike Framework is an Model-View-Controller architecture framework, where View is realized by HTML templates,
Controller is realized by components, controllers or partials, and
Model is realized by local database model and controller, component and partials data.
Spike Framework includes such a features like automatic jQuery selectors, imports, template engine, GStrings, routing, translations manager, debugging,
simple configuration, controllers, modals, components, partials, enums, globals, services, utils, REST providers, REST mocking, interceptors, local SQLLite
and WebSQL support with by-model definition, database model, database query builder and CRUD, events handling, custom implementations and overrides, screens transitions, wrapping Cordova plugins.
The main concept is simply and fast web and mobile application development with plain JavaScript and jQuery (no matter which version).
Spike Framework has only this functionality which is needed and can be done with minimalist impact on performance or code size or complicity.
Also the concept is to create abstract layer for developer to keep project consistent with minimal effort and to keep biggest freedom in implementation.
Spike Framework is in opposite to big frameworks as Ionic or Angular, which ones require extensive knowledge.
