Debug

Spike application supports basic debugging functions used as opposites to console.log
It's recommended to use build-in function because messages printed by it can be easily switched off using app.config in config.js file

Example usage debug by passing array

                app.debug('Invoke function {0} with param: {1}', ['fooFunction', arg]);

JavaScript console would print:

20:54:43 Spike Framework: Invoke function "fooFunction" with param: 100

Also you can pass object instead of array:


                app.debug('Invoke function {name} with id: {id}', {
                    name: 'fooFunction2', id: 200
                });

JavaScript console would print:

20:54:43 Spike Framework: Invoke function "fooFunction2" with id: 200

Debug methods

app.obj
Prints JavaScript object on console
Shows only when app.config.showObj is setted to TRUE
Arguments:
  • jsObject JavaScript object
Example usage: 
                        app.obj({
                            name: 'Jack',
                            surname: 'Strong'
                        });
app.log
Prints logging BLUE message with binded params on console
Shows only when app.config.showLog is setted to TRUE
Arguments:
  • logMessage String message
  • logData JavaScript object or array of params
Example usage with different data: 

                      app.log('Function result is {0}', [100]);

                      app.log('Function result is {result}', {
                            result: 100
                      });
Result on console would be:

20:54:43 Spike Framework: Function result is 100

20:54:43 Spike Framework: Function result is 100
