Configuration file

Configuration file called config.js is designed to store custom parameters used in whole application.

You can extend app.config with own Spike constants values as well as own constants. 

                app.config.extend({
                    mobileRun: true, //Spike owned
                    debug: true, //Spike owned
                    myOwnProperty: 'FACEBOOK_API_KEY' //User owned
                });

Spike constants

routing
Contains routing object from router factory. If routingEnabled is true then routing should be defined.
Default NULL
Example: 
                        app.config.extend({
                           routing: app.router.create()
                            .path('/home', {
                                controller: 'Home'
                            })
                            .other({
                                controller: 'Home'
                            })
                        });
For more info and examples see routing page
routingEnabled
Enables or disables routing in Spike application.
Default TRUE
Spike application can work in two ways: with routing and without it. It's recommended to use routing because when it's not defined, when user reload page then application will start again. Of course it can be normal behaviour on mobile. For web development routing should be enabled and defined.
mobileRun
Enables or disables running Spike application as on mobile device. If is TRUE then database initialization uses SQLLite Cordova plugin rather than WebSQL. If you testing application via Chrome Browser, then should be set to FALSE If you deploy application to the device, then should be set to TRUE
Default FALSE
showDebug
Enables or disables debug console output from Spike framework and application. Debug messages are created from app.debug(msg) command. For more info about debug see debug page
Default TRUE
showObj
Enables or disables debug console output from Spike framework and application. Debug messages are created from app.obj(msg) command. For more info about debug see debug page
Default TRUE
showWarn
Enables or disables debug console output from Spike framework and application. Debug messages are created from app.warn(msg) command. For more info about debug see debug page
Default TRUE
showError
Enables or disables debug console output from Spike framework and application. Debug messages are created from app.error(msg) command. For more info about debug see debug page
Default TRUE
showLog
Enables or disables debug console output from Spike framework and application. Debug messages are created from app.log(msg) command. For more info about debug see debug page
Default TRUE
mainController
Stores name of main controller rendered when Spike application starting (when page is refreshed)
Default Home
initialView
Sets plain (without Spike framework support) template name (path) before whole application start initializing Can be used as loading screen, splash screen etc.
Default NULL Example: 
                        app.config.extend({
                            initialView: 'app/loading.html'
                        });
rootPath
Defines application main path (ex. project src/app) Also sets root templates path used to finding template in templates.js
Default app Example: 
                        app.config.extend({
                            rootPath: 'src/app'
                        });
lang
Defines Spike application initial language
Default en Example: 
                        app.config.extend({
                            lang: 'pl'
                        });
dbMode
Defines Spike application local SQLLite database mode. Available modes: 'none' - database won't be created 'create' - creates database once and never drop 'create-drop' - drop and create database every time when application is reloaded (started again, browser reload etc)
Default create-drop Example: 
                        app.config.extend({
                            dbMode: 'create'
                        });
dbTestMode
Defines if local database should be filled with test data defined in dbTestScript
Default TRUE
dbTestScript
Defines SQL script containing TEST DATA with is executed on local database after it's creation. Script executes only when dbTestMode is setted to true
Default sql/test.sql Example: 
                        app.config.extend({
                           dbTestScript: 'sql/testScript.sql'
                        });
dbProductionScript
Defines SQL script containing PRODUCTION DATA with is executed on local database after it's creation.
Default sql/production.sql Example: 
                        app.config.extend({
                           dbTestScript: 'sql/productionScript.sql'
                        });
dbName
Defines local database name.
Default local_db Example: 
                        app.config.extend({
                            dbName: 'local_db_version2'
                        });
componentDirectory
Defines main directory containing components directories structure Ex.
  • app/component
    • topNavbar
    • bottomTabs

Default component Example: 
                        app.config.extend({
                             componentDirectory: 'components',
                        });
controllerDirectory
Defines main directory containing controllers directories structure Ex.
  • app/controller
    • homePage
    • personList

Default controller Example: 
                        app.config.extend({
                             controllerDirectory: 'controllers',
                        });
modalDirectory
Defines main directory containing modals directories structure Ex.
  • app/modal
    • loginModal
    • confirmModal

Default modal Example: 
                        app.config.extend({
                              modalDirectory: 'modals'
                        });
partialDirectory
Defines main directory containing partials directories structure Ex.
  • app/partial
    • menuItems
    • loginForm

Default partial Example: 
                        app.config.extend({
                              partialDirectory: 'partial'
                        });
transitions
Defines if transitions between controllers should be enabled For default implementation of transitions see overriding transitions
Default FALSE Example: 
                        app.config.extend({
                              transitions: true
                        });
transitionAnimation
Contains default implementation of transition function For overriding see overriding transitions
Default FALSE Example: 
                        app.config.extend({
                               transitionAnimation: function(oldViewSelector, newViewSelector, appStartup, fromController, toController, complete){
                                    ...
                               }
                        });
