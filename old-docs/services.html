Service module

Spike Framework divide application into separately modules and one of them is service.

service should be used as storage for business logic and business data

Services are singletons

Services preferable directory structure should be as follows:

  • src/app (value from app.config.rootPath)
    • service
      • authorization.service.js
      • customer.service.js

Also naming convention should be keeped as:

Service file name should be named SERVICE_NAME.service.js

Example declaration of service:

                'import $this as app.service.Customer';

                app.service.register("Customer", {

                    __customers: [],

                    getCustomers: function(){
                        return $this.__customers;
                    }

                });

Then service can be used with static reference as well in imports in any another modules: 

               'import $customerService as app.service.Customer'; //import way

                app.service.Customer //static way
