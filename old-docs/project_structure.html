Project structure

Spike Framework forces developer to keep consistent MVC architecture in project by using strict naming conventions for directories and files, even it's extensions.

Below is presented example project tree:

  • app
    • component
      • personList
        • personList.component.js
        • personList.view.html
        • noResult.template.html
    • controller
      • home
        • home.controller.js
        • home.view.html
    • modal
      • loginModal
        • loginModal.modal.js
        • loginModal.view.html
    • partial
      • loginForm
        • loginForm.partial.js
        • loginForm.partial.html
    • service
      • person.service.js
      • auth.service.js
    • util
      • list.util.js
    • config.js
    • main.js

There is few rules creating project structure:

  • 1. component, controller, modal and partial directories can have another names but then names should be defined in app.config using componentDirectory, controller, modal and partial. Of course names must be defined in gruntfile project tree.
  • 2. Rest of names may be any but must be defined in gruntfile project tree.
  • 3. In general all html files should be named with extension view.html, for example personList.view.html
  • 4. partials templates should be named with extension partial.html, for example loginForm.partial.html
  • 5. Static templates should be named with extension template.html, for example noResult.template.html
  • 6. All javascript files should be named with extension telling which module is, for example person.service.js
